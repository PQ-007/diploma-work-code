# Бүх MDX Компонент ба Синтаксын Бүрэн Лавлах

Энэ нэг нийтлэл нь бүтээгдэхүүн болон контентын хяналт хийхэд зориулагдсан.

Үүнд:

- Одоогоор ашиглаж болох бүх custom MDX компонентууд
- Энгийн markdown синтакс (гарчиг, жагсаалт, хүснэгт, эшлэл, код, холбоос, зураг гэх мэт)
- Компонент бүрийг хэрхэн ашиглахыг тайлбарласан богино жишээнүүд

Энийг checklist болгон ашиглаад аль компонентыг үлдээх, сайжруулах, эсвэл устгахаа шийдээрэй.

---

## Хурдан Import Жишээ

Хэрэв таны MDX runtime компонентуудыг аль хэдийн map хийсэн бол гараар import хийх шаардлагагүй.
Өөр тохиргоонд explicit import хэрэглэх бол синтакс нь ихэвчлэн ийм байна:

```mdx
import {
  Alert,
  Card,
  Steps,
  Tabs,
  Accordion,
  Quote,
  Timeline,
  Image,
  ImageGrid,
  VideoEmbed,
  Table,
  ComparisonTable,
  FeatureGrid,
  FileTree,
  Chart,
  DataTable,
  ProTip,
} from "@/mdx/components";
```

## Энгийн Markdown Синтакс

### Гарчгууд

#### H4 гарчгийн жишээ

##### H5 гарчгийн жишээ

###### H6 гарчгийн жишээ

### Текстийн Стайл

Энэ нь **bold текст**, _italic текст_, мөн **_bold + italic_** жишээ юм.

Мөн `const ready = true;` шиг inline code болон ~~зураасаар дарах~~ хэлбэр багтсан.

### Жагсаалтууд

- Эрэмбэгүй жагсаалтын эхний мөр
- Эрэмбэгүй жагсаалтын хоёр дахь мөр
- Эрэмбэгүй жагсаалтын гурав дахь мөр

1. Эрэмбэтэй жагсаалтын эхний мөр
2. Эрэмбэтэй жагсаалтын хоёр дахь мөр
3. Эрэмбэтэй жагсаалтын гурав дахь мөр

### Blockquote

> Энгийн markdown blockquote жишээ.
>
> Үүнийг тэмдэглэл, ишлэл, эсвэл онцлох өгүүлбэрт ашиглаж болно.

### Энгийн Markdown Хүснэгт

| Синтакс     | Ямар үүрэгтэй              | Жишээ            |
| ----------- | -------------------------- | ---------------- |
| `# Heading` | Гарчиг үүсгэнэ             | `## Хэсэг`       |
| `- item`    | Эрэмбэгүй жагсаалт үүсгэнэ | `- MDX сурах`    |
| `1. item`   | Эрэмбэтэй жагсаалт үүсгэнэ | `1. Эхний алхам` |
| ` ```ts `   | Кодын блок үүсгэнэ         | Доорх жишээг хар |

### Энгийн Кодын Блок

```typescript
type ComponentDecision = "keep" | "polish" | "remove";

interface ReviewResult {
  component: string;
  decision: ComponentDecision;
  reason: string;
}
```

## Эцсийн Тэмдэглэл

Санал болгож буй хяналтын урсгал:

1. Хэсэг бүрийг шалгаж, эхний сэтгэгдлийн чанарыг тэмдэглэ.
2. Компонент бүрийг keep, polish, эсвэл remove гэж тэмдэглэ.
3. Бодит нийтлэлд их ашиглагддаг компонентын polish ажлыг түрүүнд эрэмбэл.
4. Давхардсан үүрэгтэй эсвэл бага үнэ цэнтэй компонентыг устга.
